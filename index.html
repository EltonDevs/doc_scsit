The College Scheduling System is a comprehensive application designed to manage academic schedules efficiently. It provides role-based access control with three distinct user types: Administrators, Deans, and Instructors, each with specific permissions and functionalities.

üîë ADMINISTRATOR

Primary Role: Complete system management and oversight

Permissions:

  • Manage Rooms: Create, edit, and delete classroom/facility information
  • Manage Users: Add, modify, and remove instructor accounts
  • Manage Departments: Create and organize academic departments
  • Manage Courses: Define and maintain course offerings
  • Manage Subjects: Create and edit subject information
  • Manage Sections: Set up class sections and divisions
  • View All Schedules: Access complete schedule overview across all departments
  • System Configuration: Global settings and system parameters

üìä DEAN

Primary Role: Schedule creation and instructor assignment

  • Assign Rooms: Designate classrooms for scheduled classes
  • Manage Time Slots: Set class timing and duration
  • View Department Schedules: Access schedules within their department
  • Edit Schedules: Modify existing schedule entries
  • Conflict Resolution: Handle scheduling conflicts and overlaps

üîÑ Typical Use Cases

Scenario 1: Beginning of Semester Setup

  1. Admin creates/updates rooms, courses, subjects, and sections
  2. Admin adds new instructor accounts to the system
  3. Dean creates schedule entries for all courses
  4. Dean assigns instructors to each scheduled class
  5. Instructors log in to view their complete teaching schedule

Scenario 2: Mid-Semester Schedule Change

  1. Dean identifies need for schedule modification
  2. Dean edits the schedule (time/room/instructor change)
  3. System checks for conflicts
  4. Dean confirms changes
  5. Affected Instructors see updated schedule immediately

Scenario 3: Adding New Instructor

  1. Admin creates new instructor user account
  2. Admin sets up credentials and profile
  3. Dean assigns courses to new instructor
  4. New Instructor logs in and views assignments
